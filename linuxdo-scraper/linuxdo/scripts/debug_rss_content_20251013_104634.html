          <title>【Claude Code和Codex配置有效性测试工具】</title>
          <dc:creator><![CDATA[诗仙]]></dc:creator>
          <category>开发调优</category>
          <description><![CDATA[
            <p>【ccapi】：一个快速切换Claude Code和Codex配置的工具，一键切换URL、API_KEY、AUTH_TOKEN、MODEL、一键切换系统环境变量、一键URL延迟测速、一键API有效性测试、国际化支持。各位佬友欢迎送上小星星噢~【 <a href="https://github.com/4xian/claude-codex-api" rel="noopener nofollow ugc">github地址</a> 】</p>
<p><strong>本次更新日志 1.0.9：</strong><br>
新增对codex配置的支持，这样ccapi既可以管理claude也可以管理codex，一键切换各种配置，不过最有用的我觉得还是api有效性测试，一行命令即可知道当前所有配置在claude或codex终端中是否有效(注意: ccapi没有负载均衡的作用，只是方便管理配置和测试配置是否有效)</p>
<ul>
<li><strong>一键切换</strong> - 快速在不同Codex提供商之间切换</li>
<li><strong>自动配置</strong> - 自动更新 auth.json 和系统环境变量</li>
<li><strong>延迟测试</strong> - 快速同时测试所有提供商延迟以及API配置的有效性</li>
<li><strong>自动优选</strong> - 自动测试并切换到延迟最优的配置</li>
<li><strong>国际化支持</strong> - 支持中文和英文界面语言切换</li>
</ul>
<p><strong>1. 配置config.toml:</strong></p>
<pre><code class="lang-auto">codex默认使用的配置文件是config.toml，默认路径在：
【windows】C:\Users\Administrator\.codex\config.toml
【mac/linux】~/.codex/config.toml

**注意：使用命令前需要在config.toml文件中配置 api_key 和 model (这两个属性非官方属性，仅ccapi使用)：**

api_key: 数组格式，切换不同环境变量使用
model: 数组格式，切换不同模型使用

# 示例:
[model_providers.any]
name = "any"
base_url = "https://anyrouter.top/v1"
env_key = "any"
wire_api = "responses"
requires_openai_auth = true

--需添加下面两项才可正常切换配置--
api_key = [
  "your-api-key1",
  "your-api-key2"
]
model = [
  "gpt-5-codex",
  "gpt-5"
]

# 查看当前codex配置路径
ccapi cx set
</code></pre>
<p><strong>2. 查看可用提供商：</strong></p>
<pre data-code-wrap="bash"><code class="lang-bash">ccapi cx ls 或 ccapi codex ls
</code></pre>
<p><strong>3. 切换提供商:</strong> 切换到指定的 codex 提供商，同时可选择性的切换不同的模型和api_key</p>
<pre data-code-wrap="bash"><code class="lang-bash">注意切换的名称是model_providers后面的字段，比如[model_providers.any] 则是any，不是配置中的name字段
# 基本切换
ccapi cx use any

# 切换模型 (使用第2个模型，索引从1开始)
ccapi cx use any -m 2

# 切换api_key (使用第2个api_key，索引从1开始)
ccapi cx use any -k 2

# 同时切换模型和api_key（使用第2个模型，第2个api_key，索引从1开始）
ccapi cx use any -m 2 -k 2
</code></pre>
<p><strong>4. 测试提供商URL延迟：</strong></p>
<pre data-code-wrap="bash"><code class="lang-bash">ccapi cx ping
</code></pre>

<pre data-code-wrap="bash"><code class="lang-bash"># 测试所有的提供商的有效性

<p><strong>6. 最优路线自动切换：</strong></p>
