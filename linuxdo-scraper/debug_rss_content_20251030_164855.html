重启电脑后登录界面除了常用账户会多一个administrator账户，这个administrator账户可以直接进入<span class="spoiler"><a href="https://linux.do/t/topic/1107316/1">剧透</a></span><br>

          <title>自建 CDN 的五大能力支柱</title>
          <dc:creator><![CDATA[AxisNow]]></dc:creator>
          <category>开发调优</category>
          <description><![CDATA[
            <p>前文： <a href="https://linux.do/t/topic/994676" class="inline-onebox">我们做了个自建 CDN 平台，不知道对大家是否有帮助。</a></p>
<p>今天给各位佬友介绍我们的构建过程和背后要开发的核心能力。</p>
<hr>
<p>在多云架构盛行的今天，越来越多组织希望掌控自己的应用安全和交付，而 <strong>自建私有 CDN（Private CDN）</strong> 正在成为一种新的趋势。</p>
<p>本文将分享 <strong>AxisNow 打造企业级自建 CDN 平台的五大关键能力支柱</strong>，以及我们如何将这些大型互联网公司才有的技术平民化，并一步步构建出属于自己的私有边缘网络。</p>
<hr>
<h2><a name="p-9871167-h-2025-q1-1" class="anchor" href="https://linux.do#p-9871167-h-2025-q1-1"></a>一、从边缘代理开始：高性能与高可靠性的基础（2025 Q1）</h2>
<p>CDN 的最基本原理，本质上是一个 <strong>反向代理（Reverse Proxy）</strong>。<br>
过去十年，围绕反向代理诞生了大量不同用途的技术：负载均衡、API 网关、安全防护、加速传输等。</p>
<p>AxisNow 的产品之路，也从这里开始。<br>
我们的首个最小可行产品（MVP）是一款 <strong>可部署在 Linux 环境中的分布式边缘代理</strong>。</p>
<p>在选择底层技术栈时，我们对 <strong>性能、可靠性、扩展性、安全性</strong> 进行了全面评估。<br>
经过对比 Nginx、Traefik、Caddy、Pingora 等多种方案后，我们最终选择了 <strong>Nginx + OpenResty + Lua 堆栈</strong>——这是一条成熟、风险低且具备快速迭代能力的路径。</p>
<p>我们将其命名为 <strong>Aegis</strong>，代表我们的边缘代理内核。</p>
<hr>
<h2><a name="p-9871167-h-2025-q2-2" class="anchor" href="https://linux.do#p-9871167-h-2025-q2-2"></a>二、可扩展的功能插件：构建灵活的能力框架（2025 Q2）</h2>
<p>CDN 是一个庞大的系统概念，它涵盖了 <strong>安全、性能、访问控制、可用性、连通性</strong> 等多个层面。<br>
如何在基础代理之上，逐步扩展出这些能力，是我们设计产品时最重要的思考。</p>
<p>AxisNow 的聚焦目标非常明确：我们服务于 <strong>网站、应用程序与 API</strong>（不包括直播流）。<br>
围绕这些目标，我们构建出 <strong>可按需组合的功能插件体系</strong>。</p>
<p>在设计架构时，我们探索了两条技术路线：</p>
<ol>
<li>
<p><strong>以站点 / 域名为配置中心</strong></p>
<ul>
<li>这是传统 CDN 的典型模式（如 Cloudflare、Akamai）。<br>
每个产品线独立开发，虽然单点能力强，但一旦用户需要跨域协同，就会带来部署复杂和数据割裂的问题。</li>
<li>即使是 Cloudflare，也在不断尝试打破这种“单产品孤岛”式架构。</li>
</ul>
</li>
<li>
<p><strong>以策略 / 插件为中心的流量控制</strong></p>
<ul>
<li>我们的思路是，将流量控制抽象为统一的策略层，不论流量来自哪个站点或应用，都可由统一的代理内核处理。</li>
<li>这种设计让管理效率与自动化程度大大提高，但也带来新的挑战：如何让少量域名的用户直观地理解这些策略关系。<br>
我们正持续优化这部分的可视化与交互体验。</li>
</ul>
</li>
</ol>
<hr>
<h2><a name="p-9871167-dns-2025-q3-3" class="anchor" href="https://linux.do#p-9871167-dns-2025-q3-3"></a>三、基于 DNS 的全局流量管理：让自建网络具备互联网级调度（2025 Q3）</h2>
<p>当边缘代理部署在多个区域后，一个核心问题便出现了：<strong>如何智能地调度全局流量？</strong></p>
<p>对于多数企业来说，自建 BGP 网络成本高昂，因此 <strong>DNS 是实现策略路由的关键途径</strong>。<br>
在过去，我们不得不自己搭建权威 DNS 服务。而如今，云基础设施的成熟让我们能以更高层次的方式实现这项能力。</p>
<p>AxisNow 采用 <strong>“与第三方 DNS 集成编排”</strong> 的方案，通过插件化集成，实现：</p>
<ul>
<li>基于地理位置的就近访问</li>
<li>基于策略的动态地址选取</li>
<li>与监控联动的自动切换</li>
</ul>
<p>这一设计充分继承了我们在 <strong>安全 CDN 与大型分发系统（如 TikTok 级架构）</strong> 的实践经验，<br>
让企业即使在自托管场景下，也能构建稳定高效的全球流量管理系统。</p>
<hr>
<h2><a name="p-9871167-h-2025-q3-4" class="anchor" href="https://linux.do#p-9871167-h-2025-q3-4"></a>四、性能与可用性监控：让系统具备自我洞察力（2025 Q3）</h2>
<p>任何 CDN 系统都离不开监控体系。<br>
AxisNow 的监控体系聚焦在两个层面：</p>
<ol>
<li><strong>可用性监测</strong>：对边缘节点、IP、域名进行拨测，提供实时的服务可达性评估。</li>
<li><strong>时延与质量洞察</strong>：收集连接时延与请求质量，辅助 DNS 做出更优的流量调度决策。</li>
</ol>
<p>我们延续 <strong>“松耦合设计”</strong> 理念，<br>
用户既可自托管拨测节点，也可使用 AxisNow 提供的托管监控服务。</p>
<hr>
<h2><a name="p-9871167-h-2025-q1-5" class="anchor" href="https://linux.do#p-9871167-h-2025-q1-5"></a>五、可观察性与平台基础：自建体系的运维中枢（2025 Q1）</h2>
<p>自建 CDN 意味着完全掌控，因此必须拥有强大的 <strong>可观察性（Observability）</strong>。</p>
<p>AxisNow 提供了一整套监控与分析工具，帮助技术团队理解系统运行状态、追踪问题、分析流量指标与趋势：</p>
<ul>
<li><strong>实体监控</strong>：包括边缘节点、资源、插件等对象的实时状态</li>
<li><strong>全局流量分析</strong>：从资源或边缘维度查看流量分布</li>
<li><strong>事件与操作日志</strong>：包括租户操作记录与系统事件记录</li>
<li><strong>原始日志采集与外部数据仓库同步</strong>：支持将日志导出至客户自有的数据平台进行深度分析</li>
</ul>
<p>在平台层面，我们还构建了 <strong>多租户 + API First 的 SaaS 框架</strong>，<br>
支持数据隔离、灵活扩展和自定义自动化，构成了整个 AxisNow 管理面的基础。</p>
<p>AxisNow 的 API 优先设计可以轻松与您的技术栈集成，使您能够以编程方式进行部署和维护。</p>
<hr>
<h2><a name="p-9871167-h-6" class="anchor" href="https://linux.do#p-9871167-h-6"></a>下一步计划</h2>
<p>在完成基础五大支柱建设后，我们的下一阶段目标是：</p>
<ul>
<li><strong>Origin Shield（起源之盾）</strong>：减少源站压力，优化缓存命中</li>
<li><strong>Smart Routing（智能路由）</strong>：结合实时监控与 AI 策略，实现全局最优路径选择</li>
<li><strong>源站隧道</strong>：无缝连接到您的家庭网络、云内网，实现零信任网络与内网应用公开</li>
</ul>
<p>这些能力将进一步强化 AxisNow 的 <strong>私有边缘平台（Private Edge Platform）</strong>，<br>
帮助企业以 <strong>最小代价、最高控制力</strong> 构建安全高效的应用交付网络。</p>
<hr>
<h2><a name="p-9871167-h-7" class="anchor" href="https://linux.do#p-9871167-h-7"></a>结语</h2>
<p>自建 CDN 不再只是大型互联网公司的专属能力。<br>
随着基础设施生态的成熟，<strong>中小型企业乃至个人技术爱好者</strong> 都可以通过 AxisNow 平台逐步搭建、掌控并优化自己的边缘网络。</p>
<p>AxisNow 将这些大型互联网公司才有的技术平民化，<br>
为 <strong>应用安全与交付</strong> 提供了另一种全新的选择。</p>
