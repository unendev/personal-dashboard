# 示例账号说明

## 🎭 演示模式（默认启用）

系统默认启用演示模式，**无需登录**即可体验所有功能！

### 自动登录体验

- ✅ **零摩擦访问**：访客直接打开网站即可使用
- ✅ **完整功能**：体验所有系统功能
- ✅ **预设数据**：包含真实的演示数据（待办事项、计时任务、宝藏等）
- ✅ **适合HR**：方便快速了解系统能力

## 快速登录（可选）

如果需要手动登录，系统也提供了示例账号：

### 方式一：使用示例账号快速登录按钮

1. 访问登录页面：`/auth/signin`
2. 点击页面下方的 **"使用示例账号登录"** 按钮
3. 系统会自动使用示例账号登录

### 方式二：手动输入示例账号

在登录页面输入以下信息：

- **邮箱**: `demo@example.com`
- **密码**: `demo123456`

## 示例账号特性

- ✅ 自动创建：首次访问登录页面时，系统会自动确保示例账号存在
- ✅ 快速登录：无需手动输入，一键登录
- ✅ 完整功能：拥有所有系统功能的访问权限
- ✅ 数据隔离：示例账号的数据与其他用户完全隔离

## 手动创建示例账号（可选）

如果需要手动创建或重置示例账号，可以运行以下命令：

```bash
npm run ensure-demo
```

## 注意事项

1. 示例账号仅用于演示和测试目的
2. 建议在生产环境中禁用或删除示例账号
3. 示例账号的数据可能会被定期清理

## 登录后访问

登录成功后，您将被重定向到首页 `/`，可以开始使用系统的各项功能。
