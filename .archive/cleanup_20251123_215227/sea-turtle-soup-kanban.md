# 海龟汤游戏开发看板

## 项目概述
这是一个基于LLM的Web版海龟汤游戏，旨在通过AI扮演主持人，提供更智能、更灵活的问答体验。

## 当前阶段：MVP已完成，正在进行多谜题支持

## 开发路线图

### 阶段一：内容与代码分离 (多谜题支持)
- **目标**: 将谜题内容从代码中抽离，实现多谜题管理。
- **状态**: 进行中
    - [ ] **创建谜题库**: 在 `data/stories.json` 文件中预置多个谜题。
    - [ ] **改造后端API**: 修改 `app/api/sea-turtle-soup/route.ts`，使其能根据 `storyId` 动态加载谜题。
    - [ ] **创建谜题列表API**: 新增 `app/api/stories/route.ts`，用于返回谜题列表。

### 阶段二：关卡选择界面
- **目标**: 让玩家可以看到所有谜题并选择一个开始游戏。
- **状态**: 待开始
    - [ ] **创建选择页面**: 新建 `/sea-turtle-soup/lobby` 页面。
    - [ ] **展示列表**: 在页面上展示谜题列表。
    - [ ] **链接到游戏**: 实现点击谜题后跳转到游戏页面并传递 `storyId`。

### 阶段三：高级功能 (未来展望)
- **目标**: 进一步丰富游戏体验。
- **状态**: 待开始
    - [ ] **用户出题 (UGC)**: 增加表单页面，让玩家可以创建自己的谜题并提交。
    - [ ] **多人模式**: 引入 WebSocket，实现实时互动模式。
    - [ ] **提示系统**: 设计一个更智能的提示机制。
