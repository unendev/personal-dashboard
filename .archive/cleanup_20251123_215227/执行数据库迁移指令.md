# 执行数据库迁移

## 问题
数据库表 `post_tags` 不存在，需要执行 Prisma 迁移。

## 解决方案

请在项目根目录打开终端（PowerShell 或 CMD），执行以下命令：

```bash
npx prisma db push
```

或者如果上面的命令失败，尝试：

```bash
npx prisma migrate dev --name add_post_tags
```

## 命令说明

- `npx prisma db push`: 将 Prisma schema 同步到数据库（开发环境推荐）
- `npx prisma migrate dev`: 创建迁移文件并应用（生产环境推荐）

## 执行后

迁移成功后：
1. 刷新浏览器页面
2. 尝试为帖子添加标签
3. 标签功能应该能正常工作

## 验证

迁移成功后，您应该看到类似的输出：

```
✔ Generated Prisma Client
🚀  Your database is now in sync with your Prisma schema.
```

## 如果遇到问题

如果迁移失败，请检查：
1. `.env` 文件中的数据库连接字符串是否正确
2. 数据库服务是否正在运行
3. 是否有足够的数据库权限

完成后可以删除本文件。
