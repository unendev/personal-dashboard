## 🤖 AI功能技术讲解（仅AI总结）

### 功能概述

**AI每日总结**是项目中唯一真正在使用的AI功能，使用 DeepSeek API 自动分析用户的时间记录数据。

### 技术实现细节

#### 1. 核心代码结构

```typescript
// lib/ai-service.ts

class AIService {
  private static readonly DEEPSEEK_API_URL = 'https://api.deepseek.com/chat/completions';
  private static readonly DEEPSEEK_MODEL = 'deepseek-chat';

  // 生成每日总结
  static async generateSummary(userId: string, date: string) {
    // 1. 从数据库获取任务数据
    const tasks = await TimerDB.getTasksByDate(userId, date);
    
    // 2. 数据预处理
    const analysisData = this.prepareAnalysisData(tasks);
    
    // 3. 调用 DeepSeek API
    const aiSummary = await this.callDeepSeekAPI(analysisData, date);
    
    // 4. 返回结构化结果
    return {
      summary: aiSummary.summary,
      insights: aiSummary.insights,
      totalTime: analysisData.totalTime,
      categories: analysisData.categories
    };
  }
}
```

#### 2. 数据预处理

```typescript
prepareAnalysisData(tasks) {
  // 提取关键信息
  return {
    totalTime: tasks.reduce((sum, task) => sum + task.elapsedTime, 0),
    taskCount: tasks.length,
    categories: groupByCategory(tasks),
    tasks: tasks.map(t => ({
      name: t.name,
      categoryPath: t.categoryPath,
      elapsedTime: t.elapsedTime
    }))
  };
}
```

#### 3. Prompt 设计

```typescript
const systemPrompt = `你是时间数据分析器。以精炼、客观的方式总结计时数据。

输出规则：
1. 陈述事实和数据，不使用评价性词汇
2. 使用精确的数字和百分比
3. insights为3条数据观察，每条不超过50字
4. 语言简洁、直接，类似数据报告风格

输出格式（纯JSON，无markdown）：
{
  "summary": "总体数据概述",
  "insights": ["观察1", "观察2", "观察3"]
}`;
```

**Prompt 设计原则**：
- ✅ 明确角色定位
- ✅ 具体的输出格式（JSON）
- ✅ 字数限制和风格要求
- ✅ 避免AI生成太主观的内容

#### 4. 降级策略（容错设计）

```typescript
try {
  // 优先使用 AI
  const aiSummary = await this.callDeepSeekAPI(data, date);
  return aiSummary;
} catch (error) {
  console.error('AI API failed, using fallback:', error);
  // 降级到本地规则
  return this.generateFallbackSummary(userId, date);
}
```

**降级逻辑**：
- API超时 → 本地规则生成基础总结
- API返回错误 → 本地规则
- 网络异常 → 本地规则

**面试时可以说**：
> "为了保证可靠性，我实现了降级策略。如果 DeepSeek API 调用失败，系统会自动切换到本地的规则引擎生成基础版本的总结，用户不会感受到服务完全中断。"

## 🚀 其他技术亮点

### 亮点1：递归组件架构 ⭐⭐⭐⭐⭐

**功能描述**：
- 无限层级任务嵌套
- 每个任务可独立计时
- 父任务时间自动聚合
- 支持拖拽排序

**技术实现**：

#### 数据结构
```prisma
model TimerTask {
  id       String      @id
  parentId String?     // 自关联
  parent   TimerTask?  @relation("TaskHierarchy", fields: [parentId])
  children TimerTask[] @relation("TaskHierarchy")
}
```

#### 递归算法
```typescript
// 递归查找任务
const findTaskRecursive = (tasks: Task[], id: string): Task | null => {
  for (const task of tasks) {
    if (task.id === id) return task;
    if (task.children) {
      const found = findTaskRecursive(task.children, id);
      if (found) return found;
    }
  }
  return null;
};

// 递归计算总时间
const calculateTotalTime = (task: Task): number => {
  let total = task.elapsedTime;
  if (task.children) {
    task.children.forEach(child => {
      total += calculateTotalTime(child);
    });
  }
  return total;
};
```

**面试时可以说**：
> "这个递归组件是我花时间最多的部分。数据结构用了自关联设计，算法上写了递归查找、递归更新、递归时间聚合等函数。最难的是状态管理，每一层的状态要独立但又要能往上传递。单个组件就有1200+行代码。"

### 亮点2：乐观更新机制 ⭐⭐⭐⭐

**目的**：提升用户体验，操作立即反馈

**对比**：
| 传统方案 | 乐观更新 |
|---------|---------|
| 用户操作 → 等待API → 更新UI | 用户操作 → 立即更新UI → 后台保存 |
| 延迟：500-1000ms | 延迟：< 16ms |

**实现原理**：
```typescript
async function createTask(taskData) {
  // 1. 立即创建临时任务
  const tempTask = { id: `temp-${Date.now()}`, ...taskData };
  setTasks([tempTask, ...tasks]);  // 立即更新UI
  
  try {
    // 2. 后台异步保存
    const realTask = await fetch('/api/timer-tasks', {
      method: 'POST',
      body: JSON.stringify(taskData)
    }).then(r => r.json());
    
    // 3. 用真实数据替换
    setTasks(tasks => tasks.map(t => 
      t.id === tempTask.id ? realTask : t
    ));
  } catch (error) {
    // 4. 失败回滚
    setTasks(tasks => tasks.filter(t => t.id !== tempTask.id));
    alert('创建失败，请重试');
  }
}
```

### 亮点3：移动端拖拽优化 ⭐⭐⭐⭐

**问题**：原生拖拽在手机上成功率很低（~50%）

**解决方案**：
```typescript
useSensor(TouchSensor, {
  activationConstraint: {
    delay: 100,      // 延迟100ms判断
    tolerance: 5     // 移动5px后触发
  }
})
```

**效果**：
- 成功率从 50% 提升到 95%
- 不再误触发滚动
- 加了震动反馈

**面试时可以说**：
> "移动端拖拽是我遇到的一个难点。原生的拖拽库在手机上经常失败，用户想拖拽结果变成滚动。我查阅了 @dnd-kit 的文档，调整了触摸传感器的参数，设置了100ms延迟和5px容差，成功率提升到95%以上。"
